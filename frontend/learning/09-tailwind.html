🎨 Tailwind CSS

This app uses no hand-written CSS files for components. All styling comes from Tailwind CSS — a "utility-first" CSS framework where you apply small, single-purpose classes directly in your JSX.

What Is Utility-First CSS?

Traditional CSS: you write a class name, then define what it does in a separate CSS file:

/* Traditional approach */
.submit-button {
  background-color: #4f46e5;
  color: white;
  padding: 12px 24px;
  border-radius: 12px;
  font-weight: 600;
}

Tailwind approach: you apply the styles directly with pre-defined utility classes:

// Tailwind approach — the same button
<button className="bg-indigo-600 text-white px-6 py-3 rounded-xl font-semibold">
  Submit
</button>

Instead of naming the button and writing CSS somewhere else, you just write the styles inline. There's no separate stylesheet to maintain, no risk of CSS specificity conflicts, and no unused CSS shipped to the browser.

Tailwind generates only the CSS classes you actually use, so the production bundle is very small. The utility class names follow a consistent, predictable pattern — once you learn the system, you can guess any class name.

How Tailwind Works in This Project

Tailwind is integrated via the Vite plugin (@tailwindcss/vite in vite.config.js). The import in main.jsx:

import './index.css'  ← includes @import "tailwindcss" which loads all Tailwind

Tailwind scans all your JSX files, finds every className string, and generates only the CSS for those classes. No configuration needed beyond the import.

The Naming System

Tailwind class names follow a pattern: property-value. Here are the most used categories in this project:

Colors

Colors have a name and a shade (50=lightest → 950=darkest):

// Background colors
bg-gray-950   ← very dark gray (used as page background)
bg-gray-900   ← dark gray (cards, sidebar)
bg-gray-800   ← medium-dark gray (inputs, table rows)
bg-indigo-600 ← indigo blue (primary buttons, active states)

// Text colors
text-white    ← pure white
text-gray-400 ← muted gray (secondary text)
text-gray-500 ← dimmer gray (placeholder, labels)
text-indigo-400 ← soft indigo (active nav links)
text-red-300  ← light red (error messages)

// Border colors
border-gray-700  ← subtle border
border-gray-800  ← very subtle border
border-indigo-500 ← focused input border

Opacity modifier

Add /N to any color to set opacity (0-100):

bg-indigo-600/20   ← indigo background at 20% opacity (active nav item)
bg-black/60        ← black at 60% opacity (modal backdrop)
border-indigo-800/40 ← border at 40% opacity

Spacing (padding and margin)

The number is in units of 4px (1 unit = 4px):

p-4     ← padding on all sides: 16px
px-6    ← horizontal padding (left+right): 24px
py-3    ← vertical padding (top+bottom): 12px
pt-3    ← padding-top only
m-2     ← margin: 8px
mx-auto ← horizontal margin: auto (centers the element)
mt-4    ← margin-top: 16px

Sizing (width and height)

w-full     ← width: 100%
w-64       ← width: 256px (16 * 4 = 64 * 4 = 256)
h-screen   ← height: 100vh (full viewport height)
h-8        ← height: 32px
max-w-7xl  ← max-width: 80rem (the content area limit)
max-w-md   ← max-width: 28rem (login form width)

Layout (flexbox)

Almost all layout in this project uses flexbox:

flex           ← display: flex
flex-1         ← flex-grow: 1 (take up remaining space)
flex-col       ← flex-direction: column
items-center   ← align-items: center (vertical center)
justify-center ← justify-content: center (horizontal center)
justify-between ← space-between (push items to edges)
gap-3          ← gap: 12px between flex children
shrink-0       ← flex-shrink: 0 (don't shrink even if no room)

Typography

text-sm     ← font-size: 0.875rem (14px)
text-base   ← font-size: 1rem (16px)
text-lg     ← font-size: 1.125rem (18px)
text-2xl    ← font-size: 1.5rem (24px)
font-medium ← font-weight: 500
font-semibold ← font-weight: 600
font-bold   ← font-weight: 700
truncate    ← overflow: hidden; text-overflow: ellipsis (shows "..." when text too long)
uppercase   ← text-transform: uppercase
tracking-wider ← letter-spacing: wider

Borders and Rounded Corners

border       ← border-width: 1px
border-2     ← border-width: 2px
rounded      ← border-radius: 4px
rounded-lg   ← border-radius: 8px
rounded-xl   ← border-radius: 12px (used for most cards and buttons)
rounded-2xl  ← border-radius: 16px (larger cards)
rounded-full ← border-radius: 9999px (circle)

Hover, Focus, and Disabled States

Tailwind has modifier prefixes that apply styles only in certain states:

// Hover: apply styles when mouse is over the element
hover:text-white
hover:bg-gray-800
hover:text-indigo-400
hover:bg-indigo-500

// Focus: apply when element is focused (keyboard nav / input click)
focus:outline-none        ← remove the default blue outline
focus:border-indigo-500   ← show indigo border when focused
focus:ring-1              ← add a focus ring
focus:ring-indigo-500

// Disabled: apply when element has the "disabled" attribute
disabled:opacity-60
disabled:cursor-not-allowed

// Transition: animate the change (applied alongside the changing classes)
transition-colors   ← smoothly animate color changes

Real example from LoginPage — a button that changes on hover and becomes disabled:

<button
  disabled={loading}
  className="w-full bg-indigo-600 hover:bg-indigo-500 disabled:opacity-60
             disabled:cursor-not-allowed text-white font-semibold
             rounded-xl py-3 transition-colors"
>

Dynamic classNames in React

Sometimes you need to change classes based on state. Use template literals or conditional logic in the className:

// Conditional class based on isActive (NavLink in Sidebar)
className={({ isActive }) => `
  flex items-center gap-3 px-3 py-2.5 rounded-xl text-sm transition-colors
  ${isActive
    ? 'bg-indigo-600/20 text-indigo-400 font-medium'
    : 'text-gray-400 hover:text-white hover:bg-gray-800'
  }
`}

// Conditional based on a state variable
className={`px-3 py-1.5 rounded-lg text-xs font-medium transition-colors ${
  activeStatus === s.value
    ? 'bg-indigo-600 text-white'
    : 'bg-gray-800 border border-gray-700 text-gray-400 hover:text-gray-200'
}`}

Don't build class names by concatenating strings like 'bg-' + color + '-600'. Tailwind can't detect dynamically built class names — it only finds complete class names in your source code. Always write the full class name: 'bg-indigo-600', not 'bg-' + 'indigo' + '-600'. This is why StatCard defines a full colors object mapping each color name to a complete class string.
