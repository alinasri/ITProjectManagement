🗺️ Routing

React Router is the library that makes URLs work in a Single Page App. Without it, navigating anywhere would reload the page. This lesson explains every routing concept used in this project.

What Problem Does React Router Solve?

A browser normally navigates by sending a new request to the server for each URL. But in a React SPA, we want navigation to feel instant — just swap the content area, no page reload. React Router intercepts link clicks and browser back/forward buttons, preventing the actual navigation, and instead tells React which component to render for the new URL.

React Router works by listening to the browser's History API. When you click a NavLink, React Router calls history.pushState() to change the URL in the address bar (without reloading), then tells React to re-render with the new route's component.

The Building Blocks

BrowserRouter

The outermost router component. It sets up the routing context that everything inside can use. It must wrap all routes and any component that uses routing hooks.

// App.jsx — BrowserRouter wraps the entire app
<AuthProvider>
  <BrowserRouter>       ← activates routing for everything inside
    <Routes>
      ...
    </Routes>
  </BrowserRouter>
</AuthProvider>

Routes and Route

<Routes> is a container that looks at the current URL and renders only the first <Route> whose path matches. Think of it as a switch statement for URLs.

<Routes>
  // When URL is exactly /login → render LoginPage
  <Route path="/login" element={<LoginPage />} />

  // When URL starts with /admin → render AdminDashboard
  <Route path="/admin/*" element={<AdminDashboard />} />

  // Catch-all — any other URL → redirect to /
  <Route path="*" element={<Navigate to="/" replace />} />
</Routes>

The path="*" wildcard catches anything that didn't match earlier routes — it's the 404 handler.

Navigate — Programmatic Redirect

<Navigate> is a component that, when rendered, immediately redirects the browser to a different URL. It's used for redirects in guards:

// If not logged in, show /login instead of this page
if (!user) return <Navigate to="/login" replace />;

The replace prop replaces the current history entry instead of adding a new one. This means pressing the browser's Back button won't bring you back to the page you were just blocked from — it goes further back in history.

useNavigate — Navigate From Code

When you need to navigate in response to something (like a successful login or a form submit), use the useNavigate hook:

// From LoginPage.jsx
const navigate = useNavigate();

const handleSubmit = async (e) => {
  const user = await login(username, password);
  navigate('/admin');   ← go to /admin after successful login
};

navigate('/admin') is the imperative equivalent of <Navigate to="/admin" />. Use navigate in event handlers and async code. Use <Navigate> in render logic.

Nested Routes and Outlet

Nested routes are one of the most powerful features of React Router. They let you have a persistent "shell" component (like a sidebar layout) that stays mounted while different page content loads inside it.

Here is the nested route structure from App.jsx:

// Outer route — renders ProtectedLayout (sidebar + content area)
<Route element={<ProtectedLayout />}>

  // Inner routes — rendered INSIDE ProtectedLayout via Outlet
  <Route element={<RequireAuth roles={['super_admin']} />}>
    <Route path="/admin/*" element={<AdminDashboard />} />
  </Route>

  <Route element={<RequireAuth roles={['purchase_admin', 'it_head', 'super_admin']} />}>
    <Route path="/purchases" element={<PurchasesPage />} />
  </Route>

</Route>

When the user visits /purchases:

  1. React Router matches the outer route → renders ProtectedLayout
  2. ProtectedLayout renders Layout (sidebar + a placeholder slot)
  3. React Router matches the inner route → puts PurchasesPage in that slot

That "slot" is the <Outlet /> component. It's a placeholder that says "render the matched child route here".

// components/Layout.jsx
export default function Layout() {
  return (
    <div className="flex min-h-screen bg-gray-950">
      <Sidebar />              ← always rendered, never unmounts
      <main className="flex-1">
        <Outlet />            ← child route renders HERE
      </main>
    </div>
  );
}
URL: /purchases React Router renders: ┌─────────────────────────────────────────────────────┐ │ ProtectedLayout (auth guard) │ │ └── SectionsProvider │ │ └── Layout │ │ ├── Sidebar ← stays mounted │ │ └── <main> │ │ └── <Outlet /> ← RequireAuth │ │ └── <Outlet /> ← PurchasesPage │ └─────────────────────────────────────────────────────┘ URL changes to: /tenders React Router re-renders ONLY the Outlet content: └── PurchasesPage ❌ unmounts └── TendersPage ✅ mounts Sidebar stays mounted — no flicker, no re-fetch of sections

URL Parameters — useParams

The :sectionId in a path is a URL parameter. It's a wildcard that matches any value and makes that value available in code:

// Route definition in App.jsx
<Route path="/section/:sectionId/*" element={<SectionDashboard />} />

// Inside SectionDashboard.jsx, read the sectionId from the URL
import { useParams } from 'react-router-dom';
const { sectionId } = useParams();  // e.g. '3' for /section/3/

The /* at the end of the path means "this route also handles any sub-paths". Without it, only /section/3 would match — not /section/3/personnel. With /*, the section dashboard can define its own sub-routes internally.

NavLink — Active Styling

The sidebar uses NavLink instead of a plain <a> tag. NavLink automatically knows if its destination is the current URL and lets you apply different styles when it's "active":

// components/Sidebar.jsx — the SidebarLink helper
function SidebarLink({ to, icon, label }) {
  return (
    <NavLink
      to={to}
      end                      ← only active on EXACT match
      className={({ isActive }) =>
        `flex items-center gap-3 px-3 py-2.5 rounded-xl text-sm transition-colors ${
          isActive
            ? 'bg-indigo-600/20 text-indigo-400 font-medium'  ← active styles
            : 'text-gray-400 hover:text-white hover:bg-gray-800'  ← default styles
        }`
      }
    >
      {icon}
      <span>{label}</span>
    </NavLink>
  );
}

Notice that className here receives a function instead of a string. NavLink passes { isActive } to that function, and you return the appropriate class string. The end prop prevents /admin from staying active when you're on /admin/users.

The difference between <Link> and <NavLink>: both prevent page reloads and update the URL. But NavLink additionally tracks whether it matches the current URL and gives you the isActive flag for styling. Use NavLink in navigation menus, Link for other internal links.

The /report/:token Route — A Public Route

Not all routes need authentication. The public report route is accessible by anyone with the token:

// App.jsx — outside any auth guard
<Route path="/report/:token" element={<PublicReport />} />

// PublicReport.jsx reads the token from the URL
const { token } = useParams();
// Calls the public API endpoint with that token
report.getPublic(token).then(...);
