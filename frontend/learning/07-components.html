🧩 Components

This project has a set of reusable components in src/components/. Each one is a small, focused building block that pages assemble together. This lesson explains each one.

Layout — The Persistent Shell

Layout.jsx is the outer container for all authenticated pages. It renders the sidebar on the left and a main content area on the right. The current page's content goes into <Outlet />.

// components/Layout.jsx
export default function Layout() {
  const { sections } = useSections();

  return (
    <div className="flex min-h-screen bg-gray-950">
      <Sidebar sections={sections} />
      <main className="flex-1 overflow-auto">
        <div className="max-w-7xl mx-auto px-6 py-8">
          <Outlet />   ← page content renders here
        </div>
      </main>
    </div>
  );
}

The key design decision: Sidebar is outside the Outlet. This means navigating between pages (which swaps the Outlet content) never unmounts the Sidebar. The sidebar never re-renders, never flickers.

Sidebar — Role-Based Navigation

The sidebar shows different navigation links depending on the logged-in user's role. It uses useAuth() to get the user and conditionally renders link groups:

// Simplified structure of Sidebar.jsx
export default function Sidebar({ sections }) {
  const { user, logout } = useAuth();

  return (
    <aside>
      /* Brand header */
      <div>مدیریت پروژه IT</div>

      /* Navigation — different links per role */
      <nav>
        {user?.role === 'super_admin' && (
          <>
            <SidebarLink to="/admin" label="داشبورد کلی" />
            <SidebarLink to="/purchases" label="خریدها" />
            /* ...more links, plus dynamic section links */
            {sections.map(s => <SidebarLink to={`/section/${s.id}`} label={s.name} />)}
          </>
        )}

        {user?.role === 'purchase_admin' && (
          <SidebarLink to="/purchases" label="خریدها" />
        )}
      </nav>

      /* User info + logout button */
      <div>
        <p>{user?.username}</p>
        <button onClick={handleLogout}>خروج</button>
      </div>
    </aside>
  );
}

The SidebarLink helper (a private function in the same file) wraps NavLink with the active-state styling logic. It's defined at the bottom of Sidebar.jsx and is not exported — it's only used inside that file.

Modal — Overlay Dialog

Modal.jsx is a reusable overlay. It handles:

  • Showing a semi-transparent backdrop
  • Closing when the backdrop is clicked
  • Closing on Escape key (via a useEffect event listener)
  • Rendering nothing at all when open={false}
// Usage pattern in any page
const [showModal, setShowModal] = useState(false);

<button onClick={() => setShowModal(true)}>Open</button>

<Modal
  open={showModal}
  onClose={() => setShowModal(false)}
  title="Add New Item"
>
  <!-- your form or content goes here -->
  <input type="text" />
</Modal>

The if (!open) return null; line inside Modal.jsx is important — when the modal is closed, it returns nothing and React removes it from the DOM entirely. This means state inside the modal (like partially-filled form fields) resets when the modal closes and re-opens.

ConfirmDialog — Built on Modal

ConfirmDialog is a wrapper around Modal specifically for "Are you sure?" confirmations. It shows a title, a message, and two buttons (confirm + cancel).

// Usage — ask before deleting a record
<ConfirmDialog
  open={!!deleteTarget}
  title="حذف خرید"
  message="آیا از حذف این مورد اطمینان دارید؟"
  onConfirm={handleDelete}
  onCancel={() => setDeleteTarget(null)}
  loading={deleteLoading}
/>

The pattern open={!!deleteTarget} is common: deleteTarget is either null (closed) or the ID of the record to delete (open). !! converts it to a boolean.

PageHeader — Page Title Area

A consistent header bar that appears at the top of each page. Contains the page title and an optional action button (like "Add New").

<PageHeader title="خریدها">
  <button onClick={addRow}>
    <Plus className="w-4 h-4" />  خرید جدید
  </button>
</PageHeader>

The button content is passed as children, giving the page full control over what action button appears.

FilterBar — Search + Status Filters

A search input combined with status filter buttons. The parent controls all state; FilterBar just calls the callbacks when the user interacts:

<FilterBar
  searchTerm={searchTerm}
  onSearchChange={setSearchTerm}
  statuses={STATUS_OPTIONS}
  activeStatus={statusFilter}
  onStatusChange={setStatusFilter}
/>

FilterBar is a "controlled component" — it owns no state. Everything is passed in as props. This is the right design for filter bars: the page component knows the data, so the page should own the filter state too.

StatusBadge — Colored Status Label

Renders a small pill/badge with a color and label corresponding to a status string. The mapping from status to color/label is defined in config/statusConfigs.js.

StatusHistoryTimeline — Audit Trail Display

Renders the history of status changes for a record (who changed it, when, from what to what). Used inside a Modal when the user clicks the "History" button on a row.

// PurchasesPage.jsx — open history modal
const [historyTarget, setHistoryTarget] = useState(null);
const [historyRows, setHistoryRows]     = useState([]);

// Fetch history when a record is selected
useEffect(() => {
  if (!historyTarget) return;
  purchasesApi.getHistory(historyTarget).then(r => setHistoryRows(r.data));
}, [historyTarget]);

// Show it in a modal
<Modal open={!!historyTarget} title="تاریخچه">
  <StatusHistoryTimeline rows={historyRows} />
</Modal>

The Pattern: Small, Focused, Prop-Driven

Every component in this project follows the same design principles:

  1. Single responsibility — Modal handles the overlay and close behavior. It doesn't know what's inside it.
  2. Props control behavior — open, onClose, title — the parent drives the component, not internal hidden state.
  3. No business logic — components handle display and interaction. All business logic (API calls, rules) stays in the page.
  4. Reusable — the same Modal is used for add forms, edit forms, history display, confirm dialogs. It doesn't care what's inside.
