09. Glossary

Every term used across pages 01-08, defined plainly, in alphabetical order. Use this as a reference — link back to it whenever a word feels unfamiliar.

Terms

API (Application Programming Interface)

A defined set of URLs/functions one program exposes for another program to call. Here, the backend's API is the set of /api/... routes the frontend is allowed to call.

Axios

A JavaScript library for making HTTP requests, used here instead of the browser's built-in fetch for convenience features like interceptors and a configurable base URL.

bcrypt

A password-hashing algorithm designed to be deliberately slow, making it impractical to guess passwords by brute force even if the hashed values leak.

Cascade delete (ON DELETE CASCADE)

A database rule: when a row is deleted, automatically delete any other rows that reference it via a foreign key, instead of leaving orphaned references behind.

Component (React)

A JavaScript function that returns a description of UI (in JSX). React calls it to produce real DOM elements, and re-calls it whenever its props or state change.

Context (React)

A way to share a value across many components without manually passing it down as props through every level in between. Created with createContext, read with useContext (or a custom hook wrapping it).

Cookie

A small piece of data the server asks the browser to store and automatically resend on future requests to the same site. See httpOnly cookie below for the security-hardened variant used here.

CORS (Cross-Origin Resource Sharing)

A browser security mechanism that blocks a page from one origin (e.g. localhost:5173) from freely calling an API on a different origin (e.g. localhost:3000) unless the server explicitly allows it — which is what the cors() middleware in app.js does.

Express

A lightweight Node.js library for building web servers: define a URL path and HTTP method, and Express runs your function when a matching request arrives.

Foreign key

A column in one database table that stores the id of a row in another table, creating a link between the two (e.g. a project's section_id).

Hook (React)

A function (always named use...) that lets a function component use React features like state (useState) or side effects (useEffect). Hooks can only be called inside components or other hooks.

httpOnly cookie

A cookie flagged so that JavaScript running on the page cannot read or modify it — only the browser itself can attach it to outgoing requests. Used here to store the login JWT so it can't be stolen via injected/malicious JavaScript (XSS).

Join table (many-to-many table)

A database table that exists purely to link two other tables when either side can relate to multiple rows on the other side — e.g. project_responsibles links projects and personnel.

JSON (JavaScript Object Notation)

A text format for structured data (objects, arrays, strings, numbers) that both JavaScript and virtually every other language can read/write easily. The format used for every request body and response in this app's API.

JSX

A syntax extension that lets you write HTML-like markup directly inside JavaScript, which gets compiled (by Vite, here) into plain JavaScript function calls that build UI elements.

JWT (JSON Web Token)

A signed, self-contained piece of data (here: a user's id and role) that proves identity without the server needing to store session state. The signature is checked on every request to ensure the token wasn't tampered with.

Middleware

A function that runs before a route's main handler, given the request, response, and a next() function. Used for cross-cutting concerns like parsing request bodies, checking authentication, or checking permissions.

Node.js

A runtime that executes JavaScript outside a browser, with access to the filesystem and network — what makes it possible to write a server in JavaScript at all.

ORM (Object-Relational Mapper)

A library that lets you interact with a database using objects/methods in your programming language instead of writing raw SQL (e.g. Sequelize, Prisma). This app deliberately doesn't use one — it writes SQL directly via node:sqlite.

Prepared statement

A SQL query written with placeholders (?) instead of values directly inserted into the string. The database driver safely substitutes the real values, which prevents SQL-injection attacks.

Props (React)

Values passed into a component by whoever renders it, similar to arguments passed into a function — e.g. <StatCard label="Total" value={42} />.

Proxy (dev server)

A configuration that forwards requests matching a certain path (here, anything starting with /api) from one server to another, so the browser can talk to a single origin even though two different servers are actually handling things.

REST (Representational State Transfer)

A convention for API design where a URL identifies a resource and the HTTP method (GET/POST/PUT/DELETE) describes the action to take on it.

Route / route handler

A specific combination of URL path and HTTP method, mapped to a function (the handler) that runs when a matching request arrives.

Router (React Router)

A library that watches the browser's URL and renders different components based on which path matches, without triggering a full page reload — what makes a single-page app feel like it has multiple "pages."

Single-page app (SPA)

A web app that loads one HTML page and then handles all navigation client-side in JavaScript, rather than requesting a new HTML document from the server on every link click.

Soft delete

Marking a row as deleted (e.g. an is_deleted flag set to 1) instead of physically removing it from the database, so it can still be recovered or audited later.

SQLite

A relational database engine that stores everything in a single file on disk, with no separate database server process to run — used here for simplicity, appropriate for a single-server, moderate-traffic internal tool.

State (React)

Data that a component remembers across re-renders, managed via useState. Updating state schedules React to re-render the component with the new value.

Vite

A fast frontend build tool/dev server used here instead of Create React App. Compiles JSX/modern JavaScript and CSS, provides instant hot-reload in development, and bundles everything into optimized static files for production via vite build.
